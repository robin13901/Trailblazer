Define an immutable `FocusPillState`:
```dart
@immutable
class FocusPillState {
  const FocusPillState({this.name, this.percentLabel});
  final String? name;          // null only before the very first resolve
  final String? percentLabel;  // e.g. "26.4%" ; null when region has no cache row -> show "—%" in widget
  bool get hasValue => name != null;
}
```

Build a `FocusPillNotifier extends Notifier<FocusPillState>`:
- `build()`:
  - Watch `liveCameraProvider`. On each new `LiveCamera?`, (re)start a trailing
    debounce timer (~120-180 ms — Claude's discretion; CONTEXT.md line 29 says
    the ms is open, "live + smooth + no flicker" is the requirement). Cancel any
    prior timer via `ref.onDispose` + a stored `Timer?`.
  - When the debounce fires, call an async `_resolve(camera)`.
  - Return the current `state` (start with `const FocusPillState()`), so the
    notifier holds last value between resolves.
  - IMPORTANT: use `ref.listen(liveCameraProvider, ...)` inside `build()` to
    react without rebuilding state to null. Do NOT reset state to a blank value
    on camera change — hold-last-value (RESEARCH.md line 158, anti-flicker).
- `_resolve(LiveCamera camera)`:
  - `lookup = ref.read(adminRegionLookupProvider); await lookup.ensureLoaded();`
  - Walk `fallbackLevelsFrom(camera.zoom)` (Plan 08-01): first non-null
    `regionAt(camera.latitude, camera.longitude, level)` wins (RESEARCH.md
    line 392 fallback chain, ends at level 2 Deutschland so never blank).
  - If ALL null (outside Germany): keep last value, return (never blank).
  - `final row = await ref.read(coverageCacheDaoProvider).getByRegionId(region.osmId.toString());`
  - percentLabel: if row != null && row.totalLengthM > 0 ->
    `formatPercent(coveragePercent(row.drivenLengthM, row.totalLengthM))`
    else null (region exists but no coverage computed yet -> widget shows "—%").
  - Guard against out-of-order resolves: capture a monotonically increasing
    request id at the start of `_resolve`; only commit `state` if this is still
    the latest request (prevents a slow resolve clobbering a newer one -> jitter).
  - Set `state = FocusPillState(name: region.name, percentLabel: percentLabel);`

Expose:
```dart
final focusPillProvider =
    NotifierProvider<FocusPillNotifier, FocusPillState>(FocusPillNotifier.new);
```

Plain `NotifierProvider` — NO codegen. Use `dart:async` `Timer`. Store the timer
in a field; cancel it in `ref.onDispose`. Package imports only.

Note: `regionAt` is async but sub-ms after `ensureLoaded` (RESEARCH.md line 44);
the cache read is a PK point-read. The debounce + request-id guard together give
the "smooth, no flicker on fast pan" behavior.

```dart
class FocusAreaPill extends ConsumerWidget {
  const FocusAreaPill({super.key});

  @override
  Widget build(BuildContext context, WidgetRef ref) {
    final state = ref.watch(focusPillProvider);
    final theme = Theme.of(context);
    // Hold-last-value: before the first resolve, show a neutral placeholder so
    // the pill is never blank (and GlassPill's 0-dim guard is never hit with 0
    // content). After first resolve, name + % stay put during re-resolves.
    final name = state.name ?? 'Standort';       // German neutral placeholder
    final percent = state.percentLabel ?? '—%';
    return Semantics(
      label: 'Focus area: $name, coverage $percent',
      child: GlassPill(
        padding: const EdgeInsets.symmetric(vertical: 8, horizontal: 18),
        child: Column(
          mainAxisSize: MainAxisSize.min,
          crossAxisAlignment: CrossAxisAlignment.center,
          children: [
            Text(
              name,
              maxLines: 1,
              overflow: TextOverflow.ellipsis,
              style: theme.textTheme.bodyMedium
                  ?.copyWith(fontWeight: FontWeight.w600),
            ),
            Text(
              percent,
              style: theme.textTheme.bodySmall?.copyWith(
                color: theme.colorScheme.onSurface.withValues(alpha: 0.8),
              ),
            ),
          ],
        ),
      ),
    );
  }
}
```
