<task type="auto">
  <name>Task 3: DriftBackupService impl (VACUUM INTO export + validate + wipe-and-swap restore)</name>
  <files>lib/features/settings/data/drift_backup_service.dart, lib/features/settings/data/backup_service_provider.dart</files>
  <action>
Implement `DriftBackupService implements BackupService`. CRITICAL: do NOT cache an `AppDatabase` instance â€” read `appDatabaseProvider` via `Ref.read()` on each call (Pitfall 8), so post-restore it points at the fresh DB. Constructor takes `Ref` (or the provider container's ref).

**createBackup()** (SET-07):
1. `final tempDir = await getTemporaryDirectory();` (path_provider, already a dep)
2. filename `trailblazer_backup_${_timestamp()}.trailblazer` (YYYYMMDD_HHmm)
3. `await ref.read(appDatabaseProvider).customStatement('VACUUM INTO ?', [tempFile.path]);`
4. Return `Ok(tempFile.path)`. Wrap IO in `StorageError`, DB exceptions in `DatabaseError` via try/catch -> `Err`.

**validateBackup(path)** (SET-08 pre-check) â€” use `package:sqlite3/sqlite3.dart` read-only, NOT drift (avoid premature migration):
1. `sqlite3.open(path, mode: OpenMode.readOnly)`
2. `PRAGMA integrity_check` first row != 'ok' -> `BackupInvalid('integrity_check: ...')`
3. `PRAGMA user_version` (NOT schema_version â€” Pitfall 5): `uv == 0` -> `BackupInvalid('not a Trailblazer backup')`; `uv > 4` -> `BackupInvalid('backup from a newer app version ($uv > 4)')`; `uv < 4` accepted (Drift migrates on reopen).
4. Required tables present: `{'trips','trip_points','driven_way_intervals'}` subset of `sqlite_master` table names, else `BackupInvalid('missing tables: ...')`.
5. `SqliteException` -> `BackupInvalid('not a valid SQLite file: ...')`. `finally { db?.dispose(); }`. Return `BackupValid(schemaVersion: uv)`.
Use a `const kCurrentSchemaVersion = 4` (or read `AppDatabase.schemaVersion`).

**restore(path)** (SET-08 wipe-and-swap) â€” EXACT sequence (Pitfall 2, Pitfall 3):
1. `validateBackup(path)`; if `BackupInvalid` return `Err(StorageError('Invalid backup: <reason>'))`.
2. Safety snapshot: `VACUUM INTO` the CURRENT db to `${tempDir}/pre_restore_safety.trailblazer` (so a mid-swap failure is recoverable).
3. `await ref.read(appDatabaseProvider).close();` â€” MUST precede any file op.
4. `final docsDir = await getApplicationDocumentsDirectory();` (drift_flutter uses documents dir, Pitfall 3). Delete `app_db.sqlite`, `app_db.sqlite-wal`, `app_db.sqlite-shm` if they exist.
5. `await File(path).copy(File(p.join(docsDir.path, 'app_db.sqlite')).path);`
6. `ref.invalidate(appDatabaseProvider);` (rebuild next frame; Drift runs onUpgrade if user_version < 4).
7. Return `Ok(null)`. Wrap failures as `StorageError`/`DatabaseError`.

`backup_service_provider.dart`: `final backupServiceProvider = Provider<BackupService>((ref) => DriftBackupService(ref));` (plain Provider, codegen OFF).
  </action>

  <done>All three methods implemented with the exact close->swap->invalidate ordering; `VACUUM INTO` used for both export and safety snapshot; documents dir (not support dir) resolved for the swap.</done>
